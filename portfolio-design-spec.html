Style Notes

  • Background: large radial blur blobs — #ccff00 at 5% opacity + white at 5% opacity, blurred 100–120px

  • Selection: #ccff00 background, black text

  • Hover underlines: animated width transition from 0 → 100% via pseudo-element

3. Page-by-Page Layout Outline

Home /
A · Navigation (global)
  • Left: Name or initials (logo mark)
  • Right: Work · About · Contact (+ Journal if included)
  • Behavior: sticky, or transparent-to-solid on scroll
  • Mobile: hamburger → slide-in drawer or full-screen overlay
B · Hero
  • Eyebrow label: Role / discipline
  • Headline: Your name or defining statement (typographic treatment per chosen option)
  • Sub-copy: 1–2 sentences, approach or current focus (max 25 words)
  • Primary CTA: "View Work" → /work
  • Optional decorative: background gradient, grid lines, or image (branding-dependent)
C · Selected Work Teaser (optional)
  • 2–3 featured project tiles with thumbnail, title, year, category
  • "See all work" link → /work
D · Footer (global)
  • Copyright · Nav links · Social icons · Email · Availability status
Work / Projects /work
A · Navigation
  • Same as all pages
B · Page Header
  • Section title: "Work", "Selected Projects", or "Index"
  • Optional: project count, year range, or short philosophy statement
C · Project Grid
  • 2-column desktop, 1-column mobile
  • Each card: thumbnail (16:9 or 4:3), title, year, category tag, 1-sentence description
  • Hover: image scale-up or overlay reveal
  • Click: → /work/[slug] or external URL
D · Footer
About /about
A · Navigation
B · Bio Block
  • Large "About" heading or your name
  • 2–4 paragraphs: origin, approach, values, current focus
  • Optional profile photo (square or portrait)
C · Skills / Tools (optional)
  • Brief list or tag cloud — keep it short, no progress bars
D · Experience / Timeline (optional)
  • Minimal list: role, company/client, year range — link to PDF resume
E · Current Status (optional)
  • 1–2 sentences on current availability or focus
  • AvailabilityBadge component
F · Footer
Contact /contact
A · Navigation
B · Contact Block
  • Heading: "Get in Touch" or "Let's Work Together" or "Contact"
  • Primary: large email link (mailto:) styled as CTA
  • Contact form: Name · Email · Message · Send
  • Form states: Idle → Submitting → Success → Error
C · Social Links
  • GitHub, LinkedIn, Dribbble, Twitter/X — only what you actively use
D · Footer

Getting Started

npx create-next-app@latest my-portfolio \
  --typescript --tailwind --app

# Then:
# 1. Add Google Fonts <link> to app/layout.tsx
# 2. Copy CSS tokens into styles/globals.css
# 3. Add projects to data/projects.ts
# 4. Build Nav → Footer → Hero → pages
# 5. git init && git remote add origin <your-repo>
# 6. Push to GitHub → connect Vercel

Responsive Breakpoints

Performance Tips

  • Use next/image for project thumbnails — automatic WebP + lazy-load
  • Load Google Fonts via next/font to avoid render-blocking requests
  • Hero section: CSS animations only, no heavy JS bundles at paint time
  • Lazy-load the contact form component with next/dynamic
  • Use prefers-reduced-motion media query to disable entrance animations when needed
