# How to Build Content Maps

A **content map** represents a bird’s eye view of all possible touchpoints across product architecture, journeys, flows, customer experience and lifecycle.

In this context, a **touchpoint** is defined as any opportunity where an agent can act as a medium of interaction between a user and the brand. The key to building a content map is accurately capturing all these touchpoints and mapping them to a structured hierarchy.

## Step 3: Topics

At this stage, you should have your full feature list mapped out. Now, its time to populate **topics**, and expand them based on **content type**.

<aside>
📌

A **topic** is the **content idea** or **message theme** that agents can use to communicate with users. It’s not the message itself, but the *narrative* or *subject* your content expresses when engaging the user around a feature, touchpoint, or behavior. 

In most cases in Aampe, a **topic = a message group**.

👉 When defining topics, ask: “What are the ways an agent could communicate about this feature or experience? To whom, and in what contexts?”

</aside>

Topics help translate your product structure into engagement opportunities. Each topic represents a different way of speaking about a feature or touchpoint — whether it’s educational, contextual, or seasonal.

### **🎨 Defining Topics using Content Types**

A **content type** represents different creative expressions. It helps you explore how one topic can come to life across the many dimensions of your product: different user journeys, lifecycle stages, behavioral moments, recommendations, or even external conditions like weather, location, and other data feeds.

<aside>
💡

Use content types to help you ideate topics for each feature and sub-feature.

</aside>

You can have **different topics for the same touchpoints or feature**, each framed through different content types. For example, the same feature might inspire one message that educates all users (Foundational) and another that targets new users in a specific moment (Contextual).

> Think of content types as the creative lenses that help agents vary their communication — expressing the same topic in multiple, meaningful ways across the product experience.
> 

Content types make your map richer and more multi-dimensional. They encourage you to think beyond static feature explanations — to imagine how each topic might adapt to where the user is, what they’re doing, or what’s happening around them. This process is **creative and exploratory**, helping you build a content portfolio that feels diverse, contextual, and alive.

### 🧭 **From Core Types to Sub-Types**

Sub-types are the creative variations that bring depth and flexibility to your core content types. They help you explore how one idea — tied to a specific feature or touchpoint — can be communicated in multiple ways depending on audience, context, or surface.

Use sub-types to show range within a single concept: a feature can be taught, promoted, or reframed contextually. Sub-types are also **combinable**, allowing you to layer multiple lenses — like user lifecycle +  recommender  — to create richer, more adaptive communication.

Use sub-types as creative tools that help you generate more topics for the content portfolio.

## Step 4: Add more columns

Complete the structure of your map by adding in more columns. 

### Channels

The usual channels are PUSH, Email, SMS, WhatsApp, and In-App Surfaces. For some customers, In-App Surfaces may get granular like specific app screens, or even sections of a screen (banners, content cards, modals)

### Entry Event and Exit Event

For Contextual content, use columns Entry event and Exit event to map the exact events that will trigger that context. If the events are not available, use natural language to define the criteria.

### Content Exists?

This is a binary field, represented by a checkbox. If checked, it means content for that topic exists somewhere on the app or web and just needs to be repurposed for specific channels.

### Audience Type

You don’t need to add the exact audience definition. Just add in a rough idea of who should be eligible for this. Ultimately, a combination of audience type + the content type will determine the users who are eligible for this topic.

### Impact

Impact determines which topics should be implemented first in order to accelerate agentic learning in the first 3 months. 

<aside>
💡

Impact values can be:

- High - topics that are great for agents to learn quickly and across a wide audience! or topics that can be implemented very fast by reusing content.
*Examples: Feature Education, General Education..*
- Med - contextual topics that are narrower in context or audience, but add a lot of value and have the potential to impact metrics.
*Examples: Abandon Cart, Welcome Flows*
- Low - topics that will take a long time to implement due to technical challenges, or topics that are very seasonal in nature or restricted to a tiny audience
*Example: Limited time Sales, Recommender-based topics*
</aside>

Impact is determined by a number of factors like:

- Content type (a foundational topic is usually always a High impact topic)
- The customer’s business objectives and KPIs
- Technical feasibility e.g. whether a deeplink exists or not
- Whether the context exists or not (can it be repurposed or do we have to create from scratch)

However, this is where the customer needs our guidance and recommendation to determine which topics would have the most impact on agentic learning, and should be picked first. So it is up to us to fill up the first version of this column — and then collaborate with customers on priority.

### Other columns

You can also add columns for Deeplinks, Status, Date, POC etc. in order to make the map more actionable and usable for tracking content creation progress.
